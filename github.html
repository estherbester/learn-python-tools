Getting started on GitHub

Objectives:

  • Create a github account
  • Push/pull from github repos

Before we begin

You should already know what these git commands do

  1. git status
  2. git add
  3. git commit
  4. git log

Configure your git environment

  1. Open up Git Bash
  2. Copy the following lines into the shell prompt, one at a time (include the quotes):
  3. git config --global user.email "youremail@cars.com"
  4. git config --global user.name "your name"

Join Github

Sign up for an account at GitHub

WARNING: For our purposes, know that anything work you do on GitHub is public and viewable to everyone in the world!!*


Fork a repo from Github.

I have prepared a repository of files where we will be doing much of our work from now on. You will copy this repository into your own GitHub account, which is called forking

  1. Go to my Github repo: https://github.com/enam-cars/learn-python
  2. Locate the "fork" button and press it (see #1 in screencap below)
  3. You will now have a fork of the repo in your own account!!

Clone your fork to your working environment (your computer)

  1. Navigate to your fork, in your repo, and copy the HTTP url (see #2 in screencap).
  2. In the Git Bash shell, navigate to your User folder and type
    git clone PASTE_THE_URL_YOU_JUST_COPIED_HERE
  3. This will automatically download your fork of the GitHub repository from the cloud and save it to a new folder called "learn-python"
  4. Verify that the previous statement is true by navigating to the "learn-python" directory and see if the files in there match the ones listed in the GitHub repo

And you're done! (For now)

Track the upstream branch (enam-cars)

  1. go to the home repo (enam-cars/learn-python.git), copy the url
  2. in the shell, type 
    git remote add [enam] [your fork URL]
  3. now you are "tracking" my repo, so if I make any changes in my repo, you can pull them down. We'll learn how to do that at the end of this class.

(Optional) Git credential store

If you get sick of typing in your github password every time, you can your Github credentials in Windows.
  1. Download the Credential Store installer here
  2. In Git Bash, type git --exec-path This will give you the path to the git executable. It's probably something like C:\Program Files\Git/libexec/git-core/git
  3. In Git Bash, navigate to the folder where you saved the Credential Store installer
  4. In Git Bash, run this command: github-credential-store.exe -i [the path you just copied to clipboard]
  5. Say "Yes" when the popup asks if you want to store your credentials.
  6. If you then get an error message, let Esther or Andrew know. Otherwise, you should not see anything else happen.
  7. The next time you push something to Github, you will have to enter your username/password one last time, but after that you will no longer be prompted

Do some stuff!

Push

Let's say you made some changes in your repo: you edited a file, deleted a file, or added a new file. You need to tell git to track these changes in version control

  1. git add will add the file to your git tracker
  2. git commit will stage your changes and allow you to add comments to describe the changes you made
  3. git push will copy your changes from your local file into Github.
